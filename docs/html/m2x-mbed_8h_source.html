14  unsigned long read_ms() {
15  // In case of a timestamp overflow, we reset the server timestamp recorded.
16  // Notice that unlike Arduino, mbed would overflow every 30 minutes,
17  // so if 2 calls to this API are more than 30 minutes apart, we are
18  // likely to run into troubles. This is a limitation of the current
19  // mbed platform. However, we argue that it might be a rare case that
20  // 2 calls to this are 30 minutes apart. In most cases, we would call
21  // this API every few seconds or minutes, this won't be a huge problem.
22  // However, if you have a use case that would require 2 intervening
23  // calls to this be 30 minutes apart, you might want to leverage a RTC
24  // clock instead of the simple ticker here, or call TimeService::reset()
